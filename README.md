## Operational Modes

### Interactive Editor (Default)

```bash
ami-agent   # or: @
```

| Key | Action |
|-----|--------|
| **Enter** | Send message |
| **Alt+Enter** / **Ctrl+Enter** | Insert newline |
| **Ctrl+C** (with content) | Clear editor |
| **Ctrl+C** (empty, twice) | Exit |
| **F1** | Toggle help |

Session state persists across turns for multi-turn conversations.

### Query Mode

```bash
@ "What is the current git branch?"
```

Single-shot execution with streaming output. No session persistence. Ideal for scripting.

### Print Mode

```bash
./ami-agent --print path/to/instruction.txt < input.txt
```

Reads instruction from file, accepts stdin context. Full agent capabilities (hooks, all tools). Designed for pipeline integration.

## Security Architecture

AMI implements defense-in-depth across four layers:

```
Binary Guards (PATH-level) -> Hook Pipeline (runtime) -> Tier System (policy) -> File/Content Guards (data)
```

### Command Tier System

Commands are classified into tiers. Highest matching tier wins. Unclassified commands are **denied** (fail-closed).

| Tier | Scope | Default Action | Examples |
|------|-------|----------------|----------|
| **observe** | Read-only | allow | `ls`, `cat`, `grep`, `git status/log/diff` |
| **modify** | File changes | confirm | `touch`, `mkdir`, `cp`, `mv`, `sed`, `>` |
| **execute** | Run programs | confirm | `python`, `pytest`, `make`, `cargo`, `curl` |
| **admin** | Destructive | deny | `rm`, `sudo`, `chmod`, `git push/reset/commit` |

**Hard denies** (no override possible): command chaining (`&&`, `;`, `||`), background execution (`&`), inline python (`python -c`, `| python`), `pip` (use `uv`), `cd` (use absolute paths), direct shell invocation (`bash`, `sh`, `node`), disk destruction (`dd`, `shred`), `--no-verify`, `git rm --cached`.

Configuration: `ami/config/policies/command_tiers.yaml`

### Hook Validation Pipeline

The hook system dispatches events through a YAML-configured validator chain. All validators are fail-closed.

| Event | When | Validators |
|-------|------|------------|
| `PRE_BASH` | Before shell command | `command_tier` |
| `PRE_EDIT` | Before file modification | `path_traversal`, `edit_safety` |
| `POST_OUTPUT` | After LLM output | `content_safety` |

**Validators**:
- **CommandTierValidator** - Classifies commands into tiers, resolves actions from scope chain
- **PathTraversalValidator** - Detects `../`, URL-encoded sequences, null bytes, overlong UTF-8
- **EditSafetyValidator** - Blocks edits to sensitive files (`.env`, `.ssh/*`, `credentials.json`, etc.)
- **ContentSafetyValidator** - Blocks prohibited communication patterns in agent output

Configuration: `ami/config/hooks.yaml`

### Safety Wrappers

**git-guard** replaces the `git` binary in `.boot-linux/bin/` and cannot be bypassed because it IS the git command in PATH. It blocks:

- Destructive commands: `reset`, `checkout`, `clean`, `restore`, `rm`, `rebase`, `gc`, `prune`
- Destructive sub-commands: `stash drop`, `stash clear`, `branch -D`
- Force push: `--force`, `-f`, `--force-with-lease`
- Destructive flags: `--hard`, `--no-verify`
- Hook-bypass env vars: `SKIP`, `PRE_COMMIT_ALLOW_NO_CONFIG`
- Background `git push` (detected via `/proc/self/stat` process group comparison)

**podman-guard** similarly replaces `podman` and blocks bulk deletion, `system prune/migrate/reset`, and `image prune -a`.

### File & Content Guards

Security patterns are defined in YAML:
- `ami/config/patterns/sensitive_files.yaml` - File path patterns for protected files
- `ami/config/patterns/prohibited_communication_patterns.yaml` - Output content filters

## CI Pipeline

All git hooks are **native bash scripts** generated by `projects/AMI-CI/scripts/generate-hooks` from `.pre-commit-config.yaml`. No Python pre-commit runtime needed.

### Pre-commit (10 checks)

| Check | Description |
|-------|-------------|
| `check-unstaged` | Block unstaged changes, auto-stage |
| `ruff-format` | Code formatting |
| `ruff` | Linting with auto-fix |
| `block-sensitive-files` | Prevent committing credentials, `.env`, keys |
| `check-dependency-versions` | Strict version pinning for pyproject.toml, package.json, Dockerfiles |
| `check-banned-words` | Pattern matching for forbidden content |
| `check-file-length` | Max 512 lines per file |
| `check-init-files` | Empty `__init__.py` enforcement (security requirement) |
| `portal-preflight` | AMI-PORTAL npm preflight (when portal files change) |
| `mypy` | Type checking (excludes `projects/`) |

### Commit-msg (2 checks)

- `check-commit-message` - Enforce commit message format
- `block-coauthored` - Block co-authored-by lines in commit messages

### Pre-push (2 checks)

- `block-coauthored-history` - Full history scan for blocked patterns
- `verify-coverage` - Unit >90%, Integration >75% test coverage gates

CI check library: `projects/AMI-CI/lib/checks.sh`

## Bootstrap System

All external tools are bootstrapped into `.boot-linux/` for complete portability with no system package dependencies. Safety wrappers (git-guard, podman-guard) are installed as the actual binaries in PATH.

```bash
make install-bootstrap    # Interactive TUI to select components
make bootstrap-core       # Core only: uv, python, git, git-xet
```

Available components (24+ tools):
- **Core**: uv, python, git, git-xet (HuggingFace)
- **AI Assistants**: claude-code, gemini-cli, qwen-code
- **Development**: go, rust, sd, ansible
- **Containers**: podman (with podman-guard), kubernetes (kubectl, helm)
- **Security**: openssl, openssh, openvpn, cloudflared
- **Documents**: pandoc, texlive, pdfjam, wkhtmltopdf
- **Communication**: matrix-commander, synadm
- **Mobile**: adb

make register-extensions   # Updates ~/.bashrc
source ~/.bashrc           # Activate

## Backup System

Enterprise-grade backup with Google Drive integration and Zstandard compression.

```bash
ami-backup --setup-auth         # Authenticate with Google
ami-backup                      # Backup current directory
ami-backup --keep-local         # Keep local archive
ami-restore --interactive       # Interactive restore selector
ami-restore --list-revisions    # List available backups
ami-restore --latest-local      # Restore from local storage
```

Secondary backup to mounted drives (AMI-BACKUP). Automatic credential refresh on token expiry.

Providers support Model Context Protocol servers via `MCPServerConfig` in `ami/types/api.py`. MCP servers can be passed to `AgentConfig.mcp_servers` for connecting agents to external tool servers.

## Troubleshooting

| Problem | Fix |
|---------|-----|
| Extensions not found | `source ~/.bashrc` |
| Backup auth errors | `ami-backup --setup-auth` |
| Provider not responding | `which claude` / `make install-node-agents` |
| Pre-commit failing | `make pre-commit` to see detailed errors |
| git-guard blocking | Use `real-git` directly (`.boot-linux/bin/real-git`) if intentional |
